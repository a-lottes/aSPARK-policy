# Category: framework

Packs defining **framework- or library-specific standards**.

- [`spring`](spring/) — layering, dependency injection, configuration/secrets handling.
- [`react`](react/) — component structure, state management, accessibility (activates the `ux` lens).

> **Flat-id rule:** the category is metadata only, never part of the import id.
> ✅ `aspark:spring`  ❌ `aspark:framework/spring`

See [`../README.md`](../README.md) for pack anatomy and conventions.
